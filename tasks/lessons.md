# Lessons

- 2026-04-02: When a VM-side network stack depends on a static DHCP lease, configure it from the actual VM NIC MAC produced by `VMConfigurator`, not a guessed/default value. Detection signal: the guest gateway/DHCP identity looks wrong or the guest gets the wrong subnet/IP. Prevention rule: create the VM first, then pass the real NIC identity into the sidecar/network config.
- 2026-04-02: Do not verify mounts by grepping shell-quoted paths or assuming guest-visible `/var/...` paths appear unchanged in `mount` output on macOS. Detection signal: mounts succeed functionally but verification logs `unexpected fs type` or `MISSING`, followed by `Resource busy` on retry. Prevention rule: verify using raw paths, include `/private` aliases for macOS paths, and add a short post-mount settle check before declaring failure.
- 2026-04-07: Lint-driven refactors on the guest RPC path must preserve blocking `net.Listener` semantics and be validated with a full `dvm start` boot. Detection signal: activation succeeds, but the host agent proxy logs repeated HTTP/2 preface resets while the guest RPC daemon flaps with `accept ... resource temporarily unavailable`. Prevention rule: treat transport/socket code as behavior-sensitive, keep guest daemon logs visible through the shared state directory, and run an end-to-end start/stop cycle after any agent/vsock refactor.
- 2026-04-10: When a user prioritizes ergonomics over architectural purity, do not split a configuration model across multiple files without clear evidence it is worth the extra indirection. Detection signal: the proposed design needs an additional config file or cross-file lookup to express a single user concept. Prevention rule: first try a single-file schema with explicit typed fields and sensible defaults, then add indirection only if the simpler model demonstrably fails.
- 2026-04-10: Any harness or helper that stages files into `~/.local/state/dvm` must recreate its subdirectories after `dvm start`, because startup wipes and recreates the state dir. Detection signal: a post-boot copy into `~/.local/state/dvm/...` fails with `No such file or directory` even though the parent was created earlier in the script. Prevention rule: treat the state dir as boot-scoped runtime state, not persistent setup, and create staging directories immediately before writing into them.
- 2026-04-10: Do not rely on backslash-newline shell continuations inside nested `sh -lc '...'` strings when the command passes through multiple quoting layers. Detection signal: the guest shell reports `/bin/sh:  : command not found` even though each individual clause looks valid. Prevention rule: flatten guest command strings to single lines or generate them explicitly instead of using line continuations inside quoted shell fragments.
- 2026-04-10: When a top-level dispatcher accepts control flags plus passthrough flags, strip the control flags before invoking the child command. Detection signal: a child script rejects a mode flag like `--e2e` or `--all` that should have been consumed by the parent. Prevention rule: parse flags into distinct buckets for dispatch mode versus child passthrough, and verify the rendered command with a dry run.
- 2026-04-10: Do not treat nested inline `sh -lc '...$VAR...'` assertions as proof of guest runtime behavior when the command crosses multiple host and guest shell layers. Detection signal: a harness assertion reports an exported variable as empty even though the sourced env file and downstream behavior both show it is present. Prevention rule: use mounted probe scripts or other file-based probes for shell/runtime assertions instead of fragile nested one-liners.
